# AGENTS.md

Compact, high-signal guidance for OpenCode agents working in this repo.

## Stack

- **TanStack Start** (React 19, file-based router, RSC enabled) generated from `create-tsrouter-app` — see `.cta.json` for the exact addons (prisma, railway, ai, shadcn, better-auth, tanstack-query, compiler, form, store).
- **React Compiler** is on via `babel-plugin-react-compiler` in `vite.config.ts`. Don't manually memoize.
- **Nitro** server runtime in production (`start` = `node .output/server/index.mjs`). `nixpacks.toml` targets Railway.
- **Prisma 7** + `@prisma/adapter-pg` (pg pool). Generator outputs to `src/generated/prisma/` — never hand-edit, and it is gitignored (`/src/generated/` in `.gitignore`).
- **oxlint + oxfmt** for lint/format (not ESLint/Prettier). `oxlint` runs with `typeCheck: true` / `typeAware: true` and is the de-facto type gate.
- **Vitest 4** with jsdom + v8 coverage, 80% thresholds on branches/functions/lines/statements.
- **better-auth** with the Prisma adapter, two-factor plugin, Google social provider, and Resend for email. Handler mounted at `src/routes/api/auth/$.ts`.

## Runtime requirements

- Node `>=24.14.0` (enforced in `package.json` engines).
- pnpm `>=11.0.0 <12.0.0` — only supported PM; `pnpm-workspace.yaml` exists for hoist/build allowlist only, **there is no monorepo** (no `packages/*`).
- `prepare` runs `husky` automatically on install.

### Required verification order

After making non-trivial changes, run in this order:

1. `pnpm format` (or rely on pre-commit)
2. `pnpm lint`
3. `pnpm tsc --noEmit`  ← **no `typecheck` script exists; do not invent one**
4. `pnpm test:run`

Pre-commit (`husky`) already runs `lint` + `format:check` via `lint-staged`, but it does **not** typecheck. Commit-msg runs `commitlint` (conventional commits only — see `commitlint.config.ts` for the allowed `type-enum`).

## Strict conventions that will fail the build

- **File separation enforced by TanStack Start `importProtection` in `vite.config.ts`** (build=error, dev=error):
  - Client code may **not** import from `**/*.server.*`, `**/config/**`, `@prisma/client`, `better-auth`, or `pg`.
  - Server code may **not** import from `**/*.client.*` or `better-auth/react`.
  - Generated files (`src/generated/**`) are excluded from the rule but `src/**` is included — use the suffix convention.
- **Filename casing** (`oxlint.config.ts`):
  - Default: `kebabCase` (e.g. `login.tsx`, `use-form.ts`, `better-auth.ts`).
  - Override in `src/components/**`: `PascalCase` (e.g. `LoginForm.client.tsx`, `Avatar.tsx`).
- **`no-magic-numbers`** is on (allowed: `0`, `1`, array indexes, type indexes). Use `// oxlint-disable-next-line no-magic-numbers` sparingly, with a comment.
- **`no-console` is an error**, `no-debugger` is an error, `typescript/no-explicit-any` is an error, `typescript/consistent-type-imports` is error with `fixStyle: 'separate-type-imports'` — type imports go in their own `import type` line.
- **`func-style: 'expression'`** — prefer `const fn = () => ...` over `function fn()`.
- **`max-statements: 15`**, **`jsx-max-depth: 10`**.
- **`maxWarnings: 10`** in oxlint — more than 10 warnings fails.
- **Tabs win.** `oxfmt` is configured with `useTabs: true`; `.editorconfig` says spaces but oxfmt takes precedence. New code is tab-indented.
- **Coverage** is gated at **80%** for all four metrics in `vitest.config.ts`.

## Auth + data wiring

- `src/lib/auth.server.ts` exports the `auth` instance; it depends on `getBetterAuthConfig`, `getGoogleConfig`, `getResendConfig`, and `PrismaService.getInstance()`. All of those read+`zod.parse` env on import, so a missing env var crashes the server at startup — fix the env, not the code.
- `src/config/db.ts` `getDbConfig()` validates that **`DB_URL` matches the constructed string from `DB_USERNAME`/`DB_PASSWORD`/`DB_HOST`/`DB_PORT`/`DB_DATABASE`**. Both the connection string and the split fields are required.
- `src/prisma.server.ts` is a singleton — never `new PrismaClient()` directly.
- For server functions, use `getRequestHeaders()` from `@tanstack/react-start/server` and pass them to `auth.api.getSession({ headers })`. See `src/routes/_layout/_protected.tsx` for the canonical pattern.
- The catch-all auth route `src/routes/api/auth/$.ts` is the **only** place `auth.handler` is wired — don't add another.
- RSC pages compose server components via `CompositeComponent` / `createCompositeComponent` from `@tanstack/react-start/rsc` (see `src/routes/login.tsx` and `register.tsx`).

## Environment

All server env vars are declared in `src/env.d.ts` and required at startup. `.env` is gitignored.

```
DB_URL, DB_USERNAME, DB_PASSWORD, DB_HOST, DB_PORT, DB_DATABASE
BETTER_AUTH_URL, BETTER_AUTH_SECRET
GOOGLE_CLIENT_ID, GOOGLE_CLIENT_SECRET
RESEND_API_KEY
```

`prisma.config.ts` uses `env('DB_URL')`; the runtime `getDbConfig()` uses all six DB fields. Set both consistently.

## Things that took a while to figure out

- There is no monorepo despite `pnpm-workspace.yaml` — the file only configures `allowBuilds` and `publicHoistPattern`.
- `lint-staged` excludes `.agents/**` (this is the OpenCode local skills folder, not the `better-auth` docs that also live in `.agents/`).
- `useTheme()` (in `src/hooks/use-theme.ts`) writes the theme cookie via a server fn and then calls `router.invalidate()` — this is the canonical way to re-run root `beforeLoad`. Don't bypass it.
- `db:format` (Prisma formatter) is wired up; prefer it for edits inside `prisma/`.
- The dev server reads `.env` automatically via `vite`; Prisma commands need `dotenv -e .env` (already prefixed in every `db:*` script).

## Pencil MCP

The user has the pencil.dev desktop app installed on Windows. It is wired into opencode as a local stdio MCP server named `pencil` in the global `~/.config/opencode/opencode.json` — **do not** add a per-project `mcp` block here. The binary path is user-specific and lives in their global config.

- **When to use:** the user mentions `.pen` files, the pencil canvas, design, UI layout, components, variables, screenshots, or style guidelines. In those prompts say `use the pencil tool to …` so the LLM actually invokes it (MCPs are not auto-called).
- **Pre-condition:** the pencil desktop app must be running with the target `.pen` file open. If `opencode mcp list` shows `pencil` as disconnected, prompt the user to start the app and reopen the file — do not retry silently.
- **Tools:** `batch_design`, `batch_get` (read/mutate canvas), `get_screenshot` / `snapshot_layout` / `get_editor_state` (analyse), `get_variables` / `set_variables` (theming), `get_guidelines`, `export_nodes`. Run `opencode mcp debug pencil` to list them.
- **Verify visual changes** with `get_screenshot` after any `batch_design` mutation.
- **`.pen` files are plain text** — readable with the `read` tool, but prefer MCP tools for canvas mutations. Never hand-edit the JSON a MCP tool just produced without the user's explicit ask.
